# 数字射频战场仿真系统 - Web应用

基于Streamlit、Folium和Kimi API的现代化数字射频战场仿真系统Web应用。

## 功能特性

### 🎯 AI智能想定生成
- 集成Kimi大模型，智能生成战场想定
- 自然语言描述生成完整仿真配置
- 多维度参数自动优化

### 📡 雷达系统配置
- 多种雷达类型支持（相控阵、机械扫描、MIMO、无源）
- 可视化雷达部署和参数配置
- 实时性能分析和优化建议

### 🛰️ 目标系统配置
- 多种目标类型（战斗机、无人机、导弹、轰炸机等）
- 目标运动轨迹和电磁特性配置
- 批量目标生成和管理

### ⚙️ 仿真参数配置
- 灵活的时间和环境设置
- 多档性能配置模板
- 配置导入/导出功能

### 🚀 仿真运行控制
- 实时仿真运行控制
- 交互式监控界面
- 实时性能数据显示

### 📊 结果可视化
- 2D/3D地理视图
- 多维数据分析
- 交互式图表和报表

### 📈 性能评估
- 综合性能评分
- 详细性能分析
- AI智能评估报告

## 快速开始

### 环境要求
- Python 3.8+
- 4GB+ RAM
- 现代浏览器

### 安装步骤
1. 克隆仓库：
```bash
git clone https://github.com/your-org/digital_rf_battlefield_web.git
cd digital_rf_battlefield_web
```

2. 安装依赖：
```bash
pip install -r requirements.txt
```

3. 配置Kimi API（可选）：
   在 `.env` 文件中添加：
   ```
   KIMI_API_KEY=your_api_key_here
   ```

4. 启动应用：
```bash
python start_app.py
```

5. 打开浏览器访问：http://localhost:8501

## 常见问题

### 应用启动问题
**Q: 应用无法启动，提示端口被占用**
A: 修改端口号：`streamlit run app.py --server.port=8502`

**Q: 地图无法显示**
A: 检查网络连接，确保可以访问地图瓦片服务

### 功能使用问题
**Q: AI想定生成失败**
A: 检查Kimi API密钥是否正确，网络连接是否正常

**Q: 仿真运行缓慢**
A: 减少目标数量，调整时间步长，使用快速模式

### 技术问题
**Q: 如何扩展雷达类型？**
A: 在雷达配置页面添加新的雷达类型，实现相应的处理逻辑

**Q: 如何集成新的分析算法？**
A: 在后端实现算法，通过API接口暴露，在前端调用
